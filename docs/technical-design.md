# 语声 - 技术设计规范

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| UI框架 | Flutter | 跨平台UI，一次开发多端运行 |
| 核心逻辑 | Python | 音频处理、API调用、业务逻辑 |
| 语音识别API | 百度语音识别 | 在线识别，成本最低 |
| 离线识别 | Whisper（本地模型） | 基础离线支持 |
| 进程通信 | gRPC / HTTP | Flutter与Python进程间通信 |
| 打包工具 | Flutter + PyInstaller | 跨平台打包分发 |

## 核心模块

### 1. 音频采集模块
- 负责麦克风电芒采集和音频流管理
- 支持采样率：16kHz
- 支持格式：PCM 16bit
- 音频预处理：降噪、VAD（语音活动检测）

### 2. 语音识别模块
- 在线识别：百度语音识别API
- 离线识别：Whisper本地模型
- 自动切换在线/离线模式
- 识别结果缓存

### 3. 命令处理模块
- 语音命令解析（"换行"、"删除"等）
- 文本操作命令执行
- 系统控制命令

### 4. 词库管理模块
- 用户自定义词库
- 热词权重管理
- 词库导入导出

## 数据流

```
用户语音 → 麦克风 → 音频采集 → VAD检测 → 语音识别API/本地模型
                                              ↓
                                         识别结果文本
                                              ↓
                         ┌────────────────────┼────────────────────┐
                         ↓                    ↓                    ↓
                      命令处理              文本输出            缓存存储
                         ↓
                    文本/系统操作
```

## 安全设计

- API密钥存储在本地加密文件
- 音频数据不持久化存储
- 网络传输使用HTTPS
- 用户词库本地加密存储
